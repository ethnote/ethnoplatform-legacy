# Settings

The project settings are divided into different sections:

- [**General:**](#general) general settings for the project.
- [**Team:**](#team) manage the team members of the project.
- [**Annotations:**](#annotations) manage the annotations for the project.
- [**Danger Zone:**](#danger-zone) actions with permanent consequences.

## General

The general settings for the project are:

- Project name
- Project description
- Note Access (Who can access the fieldnotes in the project)
- Hour format (12 or 24 hour format)
- Include fieldnote instructions when downloading fieldnotes

The fieldnote access options are:

- All team members can access all fieldnotes
- Only note owners and team leaders can access fieldnotes
- Only note owners can access the note until shared with all team members
- Only note owners can access the note until shared with team leaders

If the fielnote needs to be shared, an toggle will be visible in the note.

## Team

The team section shows all the team members in the project. New team members can be invited by clicking on "Invite New Team Member" and entering their email. An invite will be sent to their email address, and they will be able to accept the invitation in the project overview.

The team members can be given the role of team leader or member.

A project at minimum needs one team leader.

## Annotations

Annotations are a specific symbol associated with a specific meaning. Annotations can be used to mark specific parts of the fieldnotes. The annotations can be created, edited, and deleted in the annotation settings. Each annotation can be given a name, a color and a symbol. By default, the following annotations are available:

- Comment (with the symbol \*)
- Quote (with the symbol ")

## Danger Zone

The danger zone contains actions that have permanent consequences. The actions are:

- Leave project (only possible if you're not the only team leader in the project)
- Delete project (all fieldnotes and attachments will be deleted. Only possible if you're the team leader)
